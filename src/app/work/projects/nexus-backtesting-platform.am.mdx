## እይታ

Nexus ጀርባ ሙከራ መድረክን ከመጀመሪያ ጀምሮ እስከ መጨረሻ አስኬድሁ፣ የ UI/UX ንድፍ ፣ ፊት-መለኪያ እና የኋላ ክፍል እንቅስቃሴን በሙሉ ተቆጣጣሪ ሆኜ። መድረኩ ስትራቴጂዎችን በታሰረ የገበያ መረጃ ላይ በውስን ይፈትሹ ዘንድ ይፈቅዳል።

## ዋና ስራዎች

- ታላቅ ዳታ ለመስመር ተስማሚ መዋቅር ንድፍ (ማሰባሰብ፣ ማከማቻ፣ ኮምፒውት) አቀናብረው አከናወንኩ።
- ሙከራዎችን ለማዋቀር እና ለማሳየት ፈጣን ፊት-መለኪያ አተገብርኩ።
- ስትራቴጂ አሰራር ፣ ውጤት ማስቀመጥ እና ትንታኔ ለማቅረብ የኋላ ክፍል አገልግሎቶችን ገነብቻለሁ።
- ማስጀመሪያ እና CI/CD ፕሮሴሶችን አቋቋመሁ።

## ቴክኖሎጂዎች

- Next.js, React
- Node.js
- PostgreSQL
- Docker በመጠቀም የታነቀ ስርጭት

## ውጤት

በታሰረ እና የሚደጋገም ሁኔታ ስትራቴጂዎችን ለመወሰን የሚያስችል ጽኑ መድረክ አቅርቤ ጨረስሁ።

## ችግኝ

ባለ ሰአታት የሚወስዱ ሙከራዎች እና በታላቅ ዳታ ላይ ዝግጁ ያልሆኑ መሳሪያዎች የተሻለ አካባቢ ያስፈልጋቸው ነበር።

## መፍትሔ

በፍጥነት ዳታ ማሰባሰብ ፣ የተመረጡ ትንታኔዎች እና የተከፈለ ኮምፒውት ላይ የተመሠረተ መስመር አቀናብረው አከናወንኩ።

## አወዳድሮች

```text
Data Ingest -> Partitioned Time-Series Storage -> Compute Cluster (worker pool) -> Results Store -> UI
```

### ቁልፍ ባህሪዎች
- ፈጣን ዳታ ማሰባሰብ እና መዋል
- የስትራቴጂ ሳንድቦክሲንግ እና የሚደጋገም አፈጻጸም
- ሀብታም ቪዥዋላይዜሽኖች (equity curves, drawdown, heatmaps)
- ብዙ ሩኖችን አብረው አንፃፅሩ እና ውጤቶችን ላክ

## መለኪያዎች
- በሚሊዮኖች መስመር የሚመዝገብ ዳታ ላይ እንኳ ከተሻለ እንቅስቃሴ ጋር ሙከራዎችን አከናወነ
- የሙከራ ጊዜን ከሰዓታት ወደ ደቂቃዎች አሳነሰ

## ማስተማር
- የሚደጋገም አፈጻጸም አስፈላጊ ነው፤ የተወሰኑ ፈተናዎች ሂደትን አመቻቹ።
- ጥሩ ካሽንግ ያለው ትንሽ ኮምፒውት ከፍተኛ ፍጥነት ይሰጣል።

## የሚመጡ እርምጃዎች
- የተበተነ ጀርባ ሙከራ ድጋፍ እና የክላውድ ኮምፒውት
